前置

DLR–前序遍历（根在前，从左往右，一棵树的根永远在左子树前面，左子树又永远在右子树前面 ）

LDR–中序遍历（根在中，从左往右，一棵树的左子树永远在根前面，根永远在右子树前面）

LRD–后序遍历（根在后，从左往右，一棵树的左子树永远在右子树前面，右子树永远在根前面）

    • T：存储数据元素的具体类型，如int、double等
    • Container：指定优先级队列底层使用的基础容器（默认会使用vector容器），我们也可以指定使用deque容器。
    • Compare：指定优先级队列中元素存储的排序规则，由该规则就可以确定每个元素的优先级高低。默认使用less<T>，即从大到小的排序规则，此时构建的是最大堆。当然，我们也可以使用greater<T>，即按照从小到大的排序规则来存储元素，此时构建的是最小堆

    子问题：反转[a,b)

    递归法总结：

    1.找出子问题

    我们要用递归逆转这个链表

    1−>2−>3−>4−>5−>6−>nullptr1->2->3->4->5->6->nullptr1−>2−>3−>4−>5−>6−>nullptr

    我们的递归作用方式是这样

    R(1−>R(2−>R(3−>R(4−>R(5−>R(6−>nullptr)))))R(1->R(2->R(3->R(4->R(5->R(6->nullptr)))))R(1−>R(2−>R(3−>R(4−>R(5−>R(6−>nullptr)))))

    我们的递归解决问题是对子问题QiQ_iQi​ 用递归R(Qi)R(Q_i)R(Qi​)解决，我们子问题是什么呢，是反转一小段链表。

    2.子问题解决方式

    对QiQ_iQi​定义解SQiS_{Q_i}SQi​​，对QiQ_iQi​定义何时终止

    举例：链表反转

    终止条件是当head == nullptr || head->next == nullptr

    子问题QiQ_iQi​解法是R(LinkList)R(LinkList)R(LinkList)中LinkListLinkListLinkList的反转，我们每一次递归是在前一次子问题结果上进行再解决的，所以我们需要有个对已经处理的子问题结果接收存储

    ListNode* last = reverse(head->next);

    当前子问题是把当前传入的节点逆序接到已经完成的子问题结果上
